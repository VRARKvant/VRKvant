# План разработки: Этап 4 (Собственная In-Browser CMS)

## Текущие задачи

- [ ] **Шаг 1: Очистка старой CMS**
  - Удалить директорию `admin/` (файлы `config.yml`, `index.html`), так как Sveltia нам больше не нужна.

- [ ] **Шаг 2: Интерфейс сохранения в GitHub (HTML/CSS)**
  - В секции `#editor` добавить скрытую или выдвижную панель "Настройки публикации".
  - Добавить поля ввода метаданных:
    - Заголовок (input)
    - Тип контента (select: Урок, Шпаргалка, Проект)
    - Трек (select, динамически подтягивать существующие)
    - Модуль (input/datalist)
    - Порядок (number)
  - Добавить поле ввода для GitHub Token (с кнопкой сохранения в `localStorage`).
  - Заменить кнопку "Скачать .md" на кнопку "Опубликовать на GitHub".

- [ ] **Шаг 3: Интеграция с GitHub REST API (`js/github-api.js`)**
  - Создать модуль `js/github.js`.
  - Написать функцию `publishToGithub(content, path, message, token)`.
  - Реализовать сборку YAML Frontmatter на основе введенных метаданных.
  - Реализовать логику: получение текущего SHA файла (если он есть), кодирование контента в Base64, отправка PUT-запроса к GitHub API.

- [ ] **Шаг 4: Улучшение визуального редактора**
  - Синхронизировать скролл: при прокрутке текста в `<textarea>`, правая панель `#editor-preview` должна скроллиться пропорционально.
  - Настроить "умную вставку": при использовании шаблонов (галерея, до/после) курсор должен оставаться внутри вставленного шорткода для быстрого редактирования путей.
